學習路徑

8 步驟建立 OCP AI 平台

依序完成以下模組，從環境準備到提供生產級 AI 服務

⚠️

前置需求

  • OpenShift Local (CRC) 或 OCP 4.14+ 叢集並具備 cluster-admin 權限
  • 已設定 eval $(crc oc-env) 環境變數
  • 至少 16 GB RAM (建議 32 GB+)、8 vCPU、100 GB 磁碟
01

環境準備與前置設定

確認 OCP/CRC 叢集正常運作，安裝必要的 CLI 工具，設定 StorageClass，並確認 cluster-admin 權限。了解 OpenShift 基礎操作。

CRC oc CLI StorageClass RBAC
02

安裝 Red Hat OpenShift AI Operator

透過 OperatorHub 安裝 RHOAI Operator，建立 DataScienceCluster CR，啟用所有核心元件，並驗證安裝狀態。

RHOAI Operator DataScienceCluster OperatorHub
03

使用者管理與專案建立

設定 OAuth Identity Provider，建立使用者群組，配置 Data Science 使用者/管理員群組，建立 Data Science Project 並分配資源配額。

OAuth / HTPasswd Groups & RBAC ResourceQuota Data Science Project
04

Jupyter Notebook 工作環境

學習如何為使用者啟動 Jupyter Notebook 伺服器，選擇運算映像（Standard Data Science、PyTorch、TensorFlow），設定 PVC 持久化儲存，並掌握 GPU 資源申請。

Jupyter PyTorch/TensorFlow PVC GPU 加速
05

Data Science Pipelines (自動化流程)

設定 Pipeline Server（需要 S3 儲存），使用 Kubeflow Pipelines SDK 建立 ML 訓練流程，並透過 RHOAI Dashboard 管理和監控 Pipeline 執行。

Kubeflow Pipelines MinIO / S3 Pipeline Server KFP SDK
06

模型部署與推論服務

使用 KServe (Single-Model) 或 ModelMesh (Multi-Model) 部署訓練好的模型，建立 InferenceService，設定 Route 提供 REST/gRPC API，並監控推論服務效能。

KServe ModelMesh InferenceService REST API
07

除錯程序與技巧 (Troubleshooting)

掌握 OpenShift AI 的診斷工作流。學習如何處理記憶體不足、Pod 啟動失敗、權限錯誤等常見問題，並學會優化資源分配。

oc describe Resource Limits DSC Management Events

08

🆕 生產環境 GPU 叢集規劃

從 CRC 開發環境邁向生產級 GPU 叢集 — Login Node + 8 × GPU Node (8×A100) 的完整架構規劃，涵蓋網路設計、GPU Operator、MIG 切割、Kueue 排程、多租戶配額與監控告警。

核心元件

RHOAI 主要功能模組

每個元件解決 MLOps 生命週期中的特定問題

🖥️

RHOAI Dashboard

統一的 Web 介面，讓使用者管理 Notebook、Pipeline、Model Serving 及資料連線，無需接觸底層 Kubernetes API。

查看安裝步驟 →
📓

Jupyter Notebook

基於 JupyterHub 的多使用者 Notebook，支援 Standard Data Science、PyTorch、TensorFlow、Code Server 等映像，可申請 GPU。

查看設定方式 →
🔄

Data Science Pipelines

基於 Kubeflow Pipelines 的 ML 工作流程引擎，使用 Python SDK 定義可重複執行的資料前處理、訓練與評估流水線。

查看 Pipeline 教學 →
🚀

KServe (模型服務)

適合大型模型與 LLM 的單模型服務框架，支援 scale-to-zero、GPU 加速、多版本流量切分，提供標準 REST/gRPC 推論 API。

查看部署方式 →
🧮

ModelMesh (多模型服務)

適合部署大量中小型模型，多模型共享同一伺服器，有效節省資源。支援 ONNX、OpenVINO、Triton、MLServer 等框架。

查看設定教學 →
🔍

TrustyAI

AI 可信度工具，提供模型偏差偵測、公平性分析、可解釋性報告，協助確保 AI 模型符合企業治理要求。

快速指令

常用 oc 指令速查

在 CRC 環境中操作 OCP AI Platform 的核心指令

🔐 登入與環境設定
# 設定 CRC 環境變數
$ eval $(crc oc-env)
# 以 admin 登入
$ oc login -u kubeadmin -p $(crc console --credentials | grep kubeadmin | awk '{print $NF}')
# 查看目前叢集資訊
$ oc cluster-info
📦 安裝 RHOAI Operator
# 建立 Operator Namespace
$ oc apply -f manifests/01-rhoai-subscription.yaml
# 監看 Operator 安裝狀態
$ oc get csv -n redhat-ods-operator
# 建立 DataScienceCluster
$ oc apply -f manifests/02-datasciencecluster.yaml
👥 使用者管理
# 建立 HTPasswd 使用者
$ htpasswd -c -B htpasswd.txt user1
# 建立 Secret
$ oc create secret generic htpasswd-secret --from-file=htpasswd=htpasswd.txt -n openshift-config
# 查看使用者列表
$ oc get users
🔍 監控 RHOAI Pod 狀態
# 查看 RHOAI 應用 Pod
$ oc get pods -n redhat-ods-applications
# 查看 DataScienceCluster 狀態
$ oc get datasciencecluster
# 取得 RHOAI Dashboard URL
$ oc get route rhods-dashboard -n redhat-ods-applications -o jsonpath='{.spec.host}'
rhods-dashboard-redhat-ods-applications.apps-crc.testing
